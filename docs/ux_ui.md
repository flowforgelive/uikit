# Система построения UI: от фундамента к страницам

Ты описываешь два связанных концепта: **Design Tokens** (фундамент) и **Atomic Design** (методология сборки). Давай разложим всё по порядку.

---

## Этап 0: 🧱 Design Tokens (Дизайн-токены)

Это **самый первый шаг** — ещё до любых элементов. Ты задаёшь «правила игры»:

### Цвета
```
Primary:       #2563EB
Secondary:     #7C3AED
Success:       #16A34A
Warning:       #EAB308
Error:         #DC2626
Neutral-100:   #F5F5F5
Neutral-200:   #E5E5E5
...
Neutral-900:   #171717
```

### Типографика (шрифты)
```
Font Family:    Inter (основной), Mono (код)

Размеры (scale):
  xs:    12px
  sm:    14px
  base:  16px
  lg:    18px
  xl:    20px
  2xl:   24px
  3xl:   30px
  4xl:   36px

Вес (weight):
  Regular:  400
  Medium:   500
  Semibold: 600
  Bold:     700

Line Height:
  tight:    1.25
  normal:   1.5
  relaxed:  1.75
```

### Сетка отступов (Spacing Scale)
> Обычно кратна **4px** или **8px** (называют «8pt grid»)

```
0:   0px
1:   4px
2:   8px
3:   12px
4:   16px
5:   20px
6:   24px
8:   32px
10:  40px
12:  48px
16:  64px
```

### Скругления (Border Radius)
```
none:  0
sm:    4px
md:    8px
lg:    12px
xl:    16px
full:  9999px  (круг/пилюля)
```

### Тени (Shadows)
```
sm:   0 1px 2px rgba(0,0,0,0.05)
md:   0 4px 6px rgba(0,0,0,0.1)
lg:   0 10px 15px rgba(0,0,0,0.1)
```

### Высота элементов (Sizing)
> Ты правильно заметил — стандартизируется!

```
Высота интерактивных элементов:
  sm:   32px
  md:   40px  ← базовая (кнопки, инпуты, селекты)
  lg:   48px

Иконки:
  sm:   16×16
  md:   20×20
  lg:   24×24
```

> **Почему это важно:** кнопка высотой 40px встаёт в ряд с инпутом 40px — всё выровнено.

## Этап 1: ⚛️ Атомы (Atoms)

Самые **мелкие, неделимые** элементы, собранные из токенов:

```
┌─────────────────────────────────────┐
│  • Кнопка (button)                  │
│  • Поле ввода (input)               │
│  • Чекбокс / Радио                  │
│  • Лейбл (label)                    │
│  • Иконка (icon)                    │
│  • Тег/бейдж (badge)               │
│  • Аватар                           │
│  • Разделитель (divider)            │
│  • Ссылка (link)                    │
│  • Переключатель (toggle)           │
└─────────────────────────────────────┘
```

Пример — **кнопка** использует токены:
```
Button/md:
  height:       40px          ← sizing token
  padding:      0 16px        ← spacing token
  font-size:    14px          ← typography token
  font-weight:  600           ← typography token
  border-radius: 8px          ← radius token
  background:   #2563EB       ← color token
  color:        #FFFFFF       ← color token
```

## Этап 2: 🧬 Молекулы (Molecules)

**Несколько атомов** объединяются в функциональный блок:

```
┌──────────────────────────────────────┐
│  • Поле формы = Label + Input + Hint │
│  • Строка поиска = Input + Button    │
│  • Карточка товара (простая)          │
│  • Пункт меню = Icon + Label         │
│  • Пагинация                          │
│  • Breadcrumbs                        │
│  • Tab item                           │
└──────────────────────────────────────┘
```

Пример:
```
┌─────────────────────────┐
│  Email *          ← Label (атом)
│ ┌─────────────────────┐ │
│ │ user@example.com    │ ← Input (атом)
│ └─────────────────────┘ │
│  Введите рабочий email  ← Hint text (атом)
└─────────────────────────┘
       = Молекула "Form Field"
```

## Этап 3: 🦠 Организмы (Organisms)

**Несколько молекул + атомов** → самостоятельная секция интерфейса:

```
┌──────────────────────────────────────┐
│  • Header (лого + навигация + поиск  │
│           + аватар)                   │
│  • Форма регистрации (несколько       │
│    полей + кнопка)                    │
│  • Карточка товара (сложная, с ценой, │
│    рейтингом, кнопкой)               │
│  • Sidebar навигация                  │
│  • Footer                             │
│  • Таблица с данными                  │
│  • Модальное окно                     │
└──────────────────────────────────────┘
```

Пример:
```
┌──────────────────────────────────────────────┐
│ 🟦 Logo    Home  About  Blog    [🔍___] 👤  │
└──────────────────────────────────────────────┘
  = Организм "Header"
  
  Состоит из:
    • Logo (атом)
    • Nav items (молекулы: Icon + Label)
    • Search bar (молекула: Input + Button)
    • Avatar (атом)
```

## Этап 4: 📄 Шаблоны (Templates)

**Каркас страницы** — организмы расставлены по сетке, но **без реального контента**:

```
┌──────────────────────────────────────────┐
│            [ Header ]                     │
├──────────┬───────────────────────────────┤
│          │                               │
│ Sidebar  │     ████  ████  ████          │

│          │                               │

│            [ Footer ]                     │
└──────────────────────────────────────────┘
```

## Итого: порядок работы

```
   ┌─────────────────────┐
   │   DESIGN TOKENS      │  ← Цвета, шрифты, отступы, размеры
   │   (фундамент)        │
   └────────┬────────────┘
            ▼
   ┌─────────────────────┐
   │   ATOMS              │  ← Кнопка, инпут, иконка
   └────────┬────────────┘
            ▼
   ┌─────────────────────┐
   │   MOLECULES          │  ← Поле формы, поиск, карточка
   └────────┬────────────┘
            ▼
   ┌─────────────────────┐
   │   ORGANISMS          │  ← Хедер, форма, секция
   └────────┬────────────┘
            ▼
   ┌─────────────────────┐
   │   TEMPLATES          │  ← Скелет страницы
   └────────┬────────────┘
            ▼
   ┌─────────────────────┐
   │   PAGES              │  ← Готовый экран с контентом
   └─────────────────────┘
```

## 💡 Практические советы

| Правило                                         | Зачем                                                |
| ----------------------------------------------- | ---------------------------------------------------- |
| **Сетка 8px**                                   | Все отступы и размеры кратны 8 → всё «дышит» ровно   |
| **Одна высота** интерактивных элементов         | Кнопка + инпут + селект в одну линию без «прыжков»   |
| **Ограниченная палитра** (5-7 цветов + оттенки) | Консистентность, нет «цветового хаоса»               |
| **Шкала шрифтов** (не произвольные размеры)     | Чёткая иерархия: заголовок > подзаголовок > текст    |
| **Именование токенов** семантическое            | `color-error` вместо `color-red` (можно менять тему) |

Эта методология называется **Atomic Design** (автор — Брэд Фрост). Если хочешь углубиться — могу рассказать подробнее про любой этап!
